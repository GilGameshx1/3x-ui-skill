## 🛡️ Безопасное использование

### Перед запуском навыка

1. **Сделайте бэкап сервера**
   ```bash
   # Создайте снапшот VPS в панели хостинга
   ```

2. **Проверьте совместимость**
   - Ubuntu 20.04+ или Debian 10+
   - Root-доступ
   - Чистая установка (рекомендуется)

3. **Сохраните SSH-ключ в надёжном месте**
   ```bash
   # После генерации ключа
   cp ~/.ssh/{nickname}_key /secure/backup/location/
   chmod 600 /secure/backup/location/{nickname}_key
   ```

4. **Запишите важные данные**
   - IP сервера
   - Root-пароль (временно)
   - Имя пользователя
   - Пароль пользователя

### Во время выполнения

- **Не прерывайте процесс** — это может привести к потере доступа
- **Следите за выводом** — проверяйте каждый шаг
- **Задавайте вопросы** — если не уверены в действии

### После завершения

1. **Проверьте доступ**
   ```bash
   ssh {nickname}
   sudo whoami  # должно вернуть root
   ```

2. **Сохраните файл-гайд**
   ```bash
   # Файл содержит все учётные данные
   cat ~/vpn-{nickname}-guide.md
   ```

3. **Сделайте бэкап базы данных**
   ```bash
   sudo ./backup-db.sh
   ```

## 📖 Рекомендации по безопасности

### 1. SSH-ключи

**Хорошо:**
```bash
# Храните ключи в безопасном месте
~/.ssh/{nickname}_key  # chmod 600
```

**Плохо:**
```bash
# Никогда не храните ключи в открытом доступе
~/Downloads/{nickname}_key  # chmod 644 ❌
```

### 2. Пароли

**Хорошо:**
- Используйте менеджер паролей (Bitwarden, KeePass)
- Генерируйте уникальные пароли для каждого сервиса
- Включите 2FA для панели 3x-ui

**Плохо:**
- Не используйте одинаковые пароли
- Не храните пароли в текстовых файлах
- Не передавайте пароли через мессенджеры

### 3. Брандмауэр

**Проверка статуса:**
```bash
sudo ufw status verbose
```

**Разрешённые порты:**
- 22/tcp — SSH
- 80/tcp — HTTP (для SSL)
- 443/tcp — HTTPS (для VLESS)

**Запрещено:**
- Не открывайте порт панели 3x-ui для всех
- Используйте SSH-туннель для доступа к панели

### 4. fail2ban

**Проверка статуса:**
```bash
sudo fail2ban-client status
sudo fail2ban-client status sshd
```

**Логи:**
```bash
sudo tail -f /var/log/fail2ban.log
```

### 5. Обновления

**Регулярно обновляйте систему:**
```bash
sudo apt update && sudo apt upgrade -y
```

**Обновляйте 3x-ui:**
```bash
sudo x-ui update
```

**Обновляйте навык:**
```bash
bash ~/.qwen/skills/3x-ui-setup/install.sh update
```

## 🚨 Признаки компрометации

### Обратите внимание, если:

- ❌ Неизвестные пользователи в системе
- ❌ Необычная сетевая активность
- ❌ Изменения в конфигурационных файлах
- ❌ Неизвестные SSH-ключи в `~/.ssh/authorized_keys`
- ❌ Высокая загрузка CPU без причины

### Что делать при компрометации

1. **Немедленно смените все пароли**
2. **Отозовите SSH-ключи и создайте новые**
3. **Проверьте логи** (`/var/log/auth.log`, `/var/log/syslog`)
4. **Запустите rollback** (если нужно)
5. **Сообщите об инциденте**
